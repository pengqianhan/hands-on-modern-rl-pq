# 第6章：DPO——绕过奖励模型的魔法

> **本章目标**：理解 DPO 如何将 RLHF 问题转化为简单的分类损失，省去奖励模型训练这一工程瓶颈。你将掌握 DPO 的数学原理和实践技巧。

## 6.1 动手：用 DPO 对齐一个"毒舌"模型

- 准备一个小型偏好数据集（问题 + 好回答 + 坏回答）
- 运行 DPO 训练，每 100 步采样生成对比
- **你将获得**：DPO 的完整训练流程——比 RLHF 简单得多，不需要奖励模型

## 6.2 观察训练过程

- 模型从"阴阳怪气"逐渐变"礼貌得体"
- 训练 Loss 下降，但生成质量是否同步提升？——Loss 低 ≠ 回答好
- β（温度参数）的影响：太大则模型不敢改变，太小则训练不稳定
- **你将思考**：DPO 简单好用，但也有它的局限

## 6.3 理论：RLHF 的工程痛点

- 标准 RLHF 的三步流程：训练奖励模型 → 用 PPO 优化策略 → 部署
- 工程痛点：
  - 奖励模型训练耗时耗卡，需要大量标注数据
  - 奖励模型容易被"hack"——模型学会钻空子而非真正改善
  - PPO 训练需要同时维护 4 个模型（Actor、Critic、Reference、Reward）
- DPO 的核心思路：**跳过奖励模型，直接从偏好对中学**
- **你将理解**：DPO 不是"更好的 RL"，而是"把 RL 问题重新表述为更简单的优化问题"

## 6.4 DPO 的数学重参数化

- Bradley-Terry 偏好模型：人类偏好 y_w > y_l 的概率 = σ(r(x, y_w) - r(x, y_l))
- 关键数学变换：从奖励函数 r 到最优策略 π 的解析关系
  - π\*(y|x) ∝ π_ref(y|x) · exp(r(x,y) / β)
- 反解奖励函数：r(x,y) = β log(π_θ(y|x) / π_ref(y|x))
- 代入 Bradley-Terry 模型，得到纯策略概率表示的损失函数：
  - L = -log σ( β log(π_θ(y_w)/π_ref(y_w)) - β log(π_θ(y_l)/π_ref(y_l)) )
- 直觉解读：让好回答的概率相对参考模型上升，让坏回答的概率相对参考模型下降
- **你将理解**：DPO 的数学核心——用策略概率的比值隐式地表示奖励

## 6.5 隐式奖励

- DPO 训练后，模型内部实际蕴含了一个奖励模型：r(x,y) = β log(π_θ(y|x) / π_ref(y|x))
- 这意味着你可以用 DPO 训练后的模型来给任意回答打分
- 隐式奖励 vs 显式奖励模型：前者"免费"得到，后者需要额外训练
- **你将理解**：DPO 不是没有奖励模型，而是把奖励模型"藏"在了策略模型内部

## 6.6 回看实践：分析 DPO 训练指标

- Reward Accuracy：模型对偏好对的判别准确率（好回答得分 > 坏回答得分的比例）
- Margin：好回答与坏回答的 log 概率差（越大说明区分越明显）
- Chosen Reward 与 Rejected Reward 的走势：两者应该逐渐拉开距离
- **你将获得**：DPO 训练质量的诊断方法

## 6.7 DPO vs PPO：何时用哪个？

| 维度 | DPO | PPO |
|------|-----|-----|
| 实现复杂度 | 低（纯分类损失） | 高（4 个模型 + 在线采样） |
| 显存需求 | 低 | 高 |
| 数据需求 | 静态偏好对 | 在线采样 + 奖励模型 |
| 适用场景 | 固定数据集、小模型对齐 | 需要动态探索的复杂对齐 |
| 上限 | 受数据质量限制 | 理论上限更高 |

- **你将获得**：在项目中选择 DPO 还是 PPO 的决策框架
