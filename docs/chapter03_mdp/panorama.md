# 全景地图：两条路线与核心困境

## 本节导读

**核心内容**

- 掌握 Value-Based 与 Policy-Based 两条路线的核心差异、优势和短板。
- 理解探索与利用如何在两条路线中以不同形式出现。
- 建立从表格方法到神经网络，再到 Actor-Critic 的整体地图。

**核心公式**

$$\pi^*(s)=\arg\max_a Q^*(s,a)$$

$$J(\theta)=\mathbb{E}_{\pi_\theta}\left[\sum_{t=0}^{\infty}\gamma^t r_t\right]$$

$$\delta = r+\gamma V(s')-V(s)$$

$$V(s)\approx f(s;\theta), \qquad Q(s,a)\approx f(s,a;\theta)$$

到目前为止，我们已经掌握了 RL 的基础工具箱：MDP 定义了游戏规则，$V(s)$ 评估了局面好坏，贝尔曼方程提供了计算价值的递归方法，DP/MC/TD 给出了三种从数据中估计价值的策略。

然后我们看到，$V(s)$ 有一个根本性的局限——它不告诉你选哪个动作。这个局限催生了两条截然不同的路线：

- **路线一（Value-Based）：** 用 $Q(s,a)$ 给每个动作打分，选最高的
- **路线二（Policy-Based）：** 跳过打分，直接优化策略 $J(\theta)$

我们还讨论了奖励函数设计——无论走哪条路线，优化的目标都取决于奖励函数的质量。稀疏奖励让学习变慢，奖励作弊让优化偏离方向，内在奖励则可以驱动探索。

本节将站在更高的视角，审视两条路线的全貌、它们各自面对的核心困境，以及后续章节将如何展开。

## 两条路线对比

|          | 路线一：Value-Based             | 路线二：Policy-Based       |
| -------- | ------------------------------- | -------------------------- | -------- |
| 核心概念 | $Q(s,a)$——给动作打分            | $J(\theta)$——直接优化策略  |
| 决策方式 | $\arg\max_a Q(s,a)$（选最高分） | 从 $\pi\_\theta(a          | s)$ 采样 |
| 代表算法 | Q-Learning → DQN                | REINFORCE → PPO            |
| 优势     | 打分准确，样本效率高            | 天然支持连续动作，擅长探索 |
| 短板     | 只能处理离散动作，不擅长探索    | 方差大，打分不够准         |

## 核心困境：探索与利用

在第 3.1 节的老虎机游戏中，我们已经亲身体验了探索与利用的矛盾。现在可以看到，这个困境在两条路线上有不同的体现：

**路线一的探索困境。** $Q$ 值最高的动作永远是"已知最好的"，但"已知最好的"未必是"真正最好的"。DQN 用 ε-贪婪来强制探索——以 $\epsilon$ 的概率随机选动作，以 $1-\epsilon$ 的概率选 $Q$ 值最高的。这是一种人工补丁，不是从算法内部自然生长出来的。举个例子：在 Atari 游戏《蒙特祖玛的复仇》中，智能体需要穿过多个房间才能找到奖励。ε-贪婪的随机探索几乎不可能碰巧穿越所有房间，导致 DQN 在这种需要长期规划的任务中表现很差。

**路线二的打分困境。** 随机性策略天然兼顾探索（总有小概率尝试新动作），但 REINFORCE 的高方差使得策略梯度估计不稳定——同一种策略跑两次，梯度可能差异显著。就像一个学生每次考试分数波动很大，很难判断他到底学得好不好。举个例子：在 CartPole 中，同样的随机策略，一局碰巧坚持了 200 步，另一局只坚持了 30 步。REINFORCE 会把第一局的梯度推到完全不同的方向——不是因为策略变了，而是因为运气不同。

这两条路线的短板恰好互补——这暗示着一个自然的想法：**能不能把两条路线拼在一起？**

## 伏笔：Actor-Critic

用路线一的方法训练一个"评委"（Critic）来评估动作好坏，再用路线二的方法训练一个"演员"（Actor）来选动作：

- **Critic（评委）** 估计价值函数 $V(s)$，回答"在状态 $s$ 下，做动作 $a$ 到底好不好？"
- **Actor（演员）** 是策略网络 $\pi_\theta(a|s)$，根据评委的打分调整行为

两者形成良性循环：评委的评分越准，演员的进步就越快；演员尝试的新动作越多，评委看到的数据就越丰富，评分也越准。

这正是 Actor-Critic 架构的核心思想——两条路线的融合。

为什么这种融合是"自然的"而不是"勉强的"？因为两条路线各自的短板恰好是对方的强项。路线一（Value-Based）擅长打分但不擅长探索——它需要人工补丁（ε-贪婪）来强迫智能体尝试新动作。路线二（Policy-Based）擅长探索但打分不准——随机性策略天然尝试新动作，但没有"评委"告诉它哪个动作到底好多少。把路线一的"打分"能力和路线二的"探索"能力拼在一起，就像给一个天赋型选手配了一个专业教练——选手（Actor）负责上场尝试各种动作，教练（Critic）在旁边记录哪些动作真的有效，反馈给选手调整策略。两者缺一不可：没有教练，选手只能凭感觉；没有选手，教练的评分表再精确也赢不了比赛。

第 6 章将详细介绍 Actor-Critic 的完整架构。

## 从表格到神经网络

到目前为止，我们讨论的所有方法都有一个隐含的前提：你能把每个状态的 $V$ 值或 $Q$ 值存在一张表格里。简单的 GridWorld 只有 16 个格子，当然没问题。但真实世界的状态空间有多大？

| 问题                             | 状态空间大小               | 表格可行？ |
| -------------------------------- | -------------------------- | ---------- |
| 老虎机                           | 1                          | 可以       |
| 井字棋                           | $\sim 20{,}000$            | 勉强       |
| 国际象棋                         | $\sim 10^{47}$             | 不行       |
| CartPole                         | $\mathbb{R}^4$（无限连续） | 不行       |
| Atari 游戏                       | $\sim 10^{33700}$          | 不行       |
| 大模型（词表 5 万，序列长 1000） | $50000^{1000}$             | 不行       |

深度强化学习（Deep RL）的做法是用神经网络来代替表格。神经网络以状态为输入，以 $Q$ 值或动作概率为输出。它不需要逐行存储每个状态的值，而是通过权重来泛化——见过相似的状态，就能猜出差不多的分数。

- 如果神经网络学的是"打分表"（输入状态，输出 $Q$ 值），它就是 Value-Based（如 DQN）
- 如果神经网络学的是"行为手册"（输入状态，输出动作概率），它就是 Policy-Based（如 REINFORCE）
- 如果同时学两者——一个网络打分，一个网络选动作——它就是 Actor-Critic（如 PPO）

本书所有算法都属于 Deep RL。

## V(s) 的后续角色

$V(s)$ 在本节只是概念性的介绍，但它将在后续章节中扮演越来越重要的角色：

- **第 5 章：** REINFORCE 用 $V(s)$ 作为基线来降低方差
- **第 6 章：** $V(s)$ 成为 Critic 网络的输出——Critic 的训练本质上就是估计 $V$。届时我们将展开介绍 DP/MC/TD 的具体实现方式
- **第 7 章：** GAE 用 $V(s)$ 计算优势函数，是 PPO 的核心组件

每次 $V(s)$ 出现，你对它的理解都会更深一层。在第 3 章，它只是一个抽象的"局面评分"。到第 5 章，它会变成降低方差的工具。到第 6 章，它会变成一个可训练的神经网络。到第 7 章，它会变成 GAE 中连接 Actor 和 Critic 的桥梁。如果你回头看，会发现 $V(s)$ 是贯穿全书的一条暗线——几乎所有核心算法都离不开它。

## 本章小结

本章是全书的理论基石。读完这一章，你现在能够：

- 用 **MDP 五元组** $\langle S, A, P, R, \gamma \rangle$ 形式化描述任何 RL 问题
- 用 **贝尔曼方程** 把"无穷未来的收益"递归地压缩成一步的计算
- 用 **DP / MC / TD** 三种方法从数据中估计价值函数，并说清它们的权衡
- 用 **$Q(s,a)$** 给动作打分、用 **$J(\theta)$** 直接优化策略，并解释两条路线各自适合什么场景
- 识别奖励设计中的常见陷阱（稀疏奖励、奖励作弊）并知道应对策略
