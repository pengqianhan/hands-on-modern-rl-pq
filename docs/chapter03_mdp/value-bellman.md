# V(s) 与贝尔曼方程

## 本节导读

**核心内容**

- 掌握状态价值函数 $V^\pi(s)$ 的定义：从某个状态出发的期望累积回报。
- 理解贝尔曼方程如何把"无穷未来"递归拆成"一步奖励 + 下一状态价值"。
- 区分贝尔曼期望方程与贝尔曼最优方程，并用简单环境动手验证。

**核心公式**

$$V^\pi(s) = \mathbb{E}_\pi\left[\sum_{k=0}^{\infty}\gamma^k r_{t+k}\mid s_t=s\right]$$

$$V^\pi(s) = \sum_{a \in \mathcal{A}}\pi(a\mid s)\left[R(s,a)+\gamma\sum_{s'\in\mathcal{S}}P(s'\mid s,a)V^\pi(s')\right]$$

$$V^*(s) = \max_a\left[R(s,a)+\gamma\sum_{s'\in\mathcal{S}}P(s'\mid s,a)V^*(s')\right]$$

上一节我们搭建了 RL 的形式化框架：MDP 五元组 $\langle \mathcal{S}, \mathcal{A}, P, R, \gamma \rangle$ 描述了游戏规则，折扣累积回报 $G_t$ 定义了总分，策略 $\pi$ 决定了行动方式。但有一个关键问题还没有回答：**在当前局面下，智能体到底处于优势还是劣势？**

你可能会说："看奖励不就行了？"——但奖励只告诉你这一步拿了多少分。CartPole 中每步都是 +1，无法区分"杆子正在缓慢倾斜"和"杆子稳如泰山"。你需要的是一个更全局的评估——不仅看眼前的奖励，还要考虑"从这个局面出发，未来预期能拿多少总分"。这正是状态价值函数 $V(s)$ 的任务，它是所有 RL 算法的理论基石。

## V(s) 的定义

$V^\pi(s)$ 的正式定义是：在状态 $s$，遵循策略 $\pi$ 行动，所能获得的期望累积回报：

$$V^\pi(s) = \mathbb{E}_\pi \left[ G_t \mid s_t = s \right] = \mathbb{E}_\pi \left[ \sum_{k=0}^{\infty} \gamma^k \, r_{t+k} \;\middle|\; s_t = s \right] \tag{3.1}$$

上标 $\pi$ 表示价值依赖于策略——同一个状态，用不同策略会得到不同价值。好策略的 $V$ 高，差策略的 $V$ 低。RL 的终极目标，就是找到让 $V$ 最高的策略 $\pi^*$。

$V(s)$ 的直觉类似棋局评估器：给定一个棋盘局面，$V(s)$ 告诉你从这步开始最终能赢的概率有多大。好的局面 $V$ 值高，差的局面 $V$ 值低。AlphaGo 的核心组件之一就是一个 $V$ 网络——给它一个棋盘，它输出"黑方赢的概率"。这个数字不是随便猜的，而是通过数百万局自我对弈训练出来的 [^3]。

以老虎机为例：只有一个状态，策略是"永远选 A"（$\pi(\text{选 A}) = 1$），每步期望回报是 0.2，不考虑折扣时：

$$V = 0.6 \times (+1) + 0.4 \times (-1) = 0.2$$

但 $V^\pi(s)$ 的定义要求把从现在到永远的所有奖励加起来——你需要知道未来每一步会发生什么。对于 CartPole 的无限连续状态空间、大模型的天文数字 token 序列组合，穷举未来是不可能的。

## 贝尔曼方程的直觉：手画一张宝藏地图

贝尔曼方程（Bellman Equation）由美国数学家理查德·贝尔曼在 1957 年提出 [^1]，它提供了一个优雅的解决方案。在看公式之前，先通过一个练习建立直觉。

假设你在一个 1×5 的走廊里寻宝：

```
┌───┬───┬───┬───┬───┐
│ S │   │   │   │ 🏆 │
└───┴───┴───┴───┴───┘
```

规则：每走一步扣 1 分（消耗体力），到达宝藏 🏆 处不扣分（游戏结束），只能往右走。**从最右边开始，往左推，填出每个格子的价值。**

```
┌────┬────┬────┬────┬───┐
│ -4 │ -3 │ -2 │ -1 │ 0 │
└────┴────┴────┴────┴───┘
  S                      🏆
```

填写思路：🏆 那格价值 0；左边一格走一步就到，扣 1 分，价值 -1；再左边走两步，扣 2 分，价值 -2……依次类推。每一步只看"这一步的代价 + 下一格的价值"，就能推出整条走廊上所有格子的价值。

现在考虑一个更有趣的场景：**如果走廊有岔路怎么办？**

```
              ┌───┬───┐
              │ -1│ 0 │ 🏆  (上方路线)
          ┌───┼───┘   │
      ┌───┤ -2│       │
  ┌───┤   │   │       │
  │ S │   │   │       │  (下方路线)
  └───┴───┴───┴───────┘
```

岔路意味着同一个格子可能有多个下一步选择。这时"下一格的价值"不再是唯一确定的——你需要考虑所有可能的目的地，按概率加权。这正是贝尔曼方程从"简单版"到"完整版"需要跨越的一步。记住这个直觉——我们马上就会看到它如何体现在公式中。

回到简单的直走廊，看第 2 格（值 -2）和第 3 格（值 -1）之间的关系：

$$\text{第 2 格的价值} = \underbrace{-1}_{\text{这一步的代价}} + \underbrace{\text{第 3 格的价值}}_{\text{走到那里之后还值多少}}$$

即 $-2 = -1 + (-1)$。每一格的价值，都等于"这一步的即时奖励"加上"下一格的价值"。没有看到未来所有的步骤——只看了一步，加上"下一格值多少"，就推出了整条走廊上所有格子的价值。这个思路就是贝尔曼方程。

### 用 Python 验证走廊寻宝

把上面的手动推导过程写成代码，验证递归关系的正确性：

```python
# 1×5 走廊寻宝：从右往左填价值
n = 5              # 走廊长度（编号 0~4）
step_cost = -1     # 每步代价
V = [0] * n

# 终点（最右格）价值为 0，从右往左递推
for i in range(n - 2, -1, -1):
    V[i] = step_cost + V[i + 1]

print("各格价值:", V)
print(f"起点价值: {V[0]}，终点价值: {V[-1]}")
```

:::output
各格价值: [-4, -3, -2, -1, 0]
起点价值: -4，终点价值: 0
:::

代码做的事和你手算的一模一样：从终点（价值 0）开始，每往左一格加一次 `step_cost`，直到起点。贝尔曼方程的递归结构让计算变得极其简单——不需要看到未来所有步骤，只看一步就够了。

## 贝尔曼期望方程

将宝藏地图的直觉推广到一般情况，贝尔曼期望方程的完整形式是：

$$V^\pi(s) = \sum_{a \in \mathcal{A}} \pi(a | s) \left[ R(s, a) + \gamma \sum_{s' \in \mathcal{S}} P(s' | s, a) \, V^\pi(s') \right] \tag{3.2}$$

逐项拆解：

| 符号            | 含义                           |
| --------------- | ------------------------------ |
| $V^\pi(s)$      | 状态 $s$ 在策略 $\pi$ 下的价值 |
| $\pi(a \| s)$   | 策略选择动作 $a$ 的概率        |
| $R(s, a)$       | 即时奖励                       |
| $\gamma$        | 折扣因子                       |
| $P(s' \| s, a)$ | 转移概率                       |
| $V^\pi(s')$     | 下一状态的价值                 |

一句话：**当前状态的价值 = 策略选各动作的概率 ×（即时奖励 + 折扣后的下一状态价值）。**

这个方程之所以重要，在于它把一个需要"看到无穷未来"的问题，转化成了一个递归结构：当前状态的价值只依赖于一步的即时奖励和下一状态的价值。Puterman [^2] 对贝尔曼方程的收敛性给出了严格的数学证明。

### 用老虎机验证

设定：A 台出奖率 60%，策略是"永远选 A"，折扣因子 $\gamma = 0.9$。

老虎机是单状态的——不管选择多少轮，每一轮面对的情况完全相同，所以 $V$ 在每一轮都相同。代入贝尔曼方程：

$$V = \underbrace{[0.6 \times (+1) + 0.4 \times (-1)]}_{= 0.2} + \underbrace{\gamma}_{0.9} \times \underbrace{V}_{\text{同一状态}}$$

方程变成关于 $V$ 的一元方程：

$$V = 0.2 + 0.9V \implies 0.1V = 0.2 \implies \boxed{V = \frac{0.2}{1 - 0.9} = 2.0}$$

$\gamma$ 对 $V$ 的影响：$\gamma = 0$（只看眼前）时 $V = 0.2$；$\gamma = 0.9$（重视未来）时 $V = 2.0$；$\gamma \to 1$ 时 $V \to \infty$（无限期游戏累积奖励发散）。

### 用 Python 验证

把上面的推导过程写成代码，看看不同 $\gamma$ 对 $V$ 的影响：

```python
# 老虎机贝尔曼方程：V = R + gamma * V  =>  V = R / (1 - gamma)
R = 0.2  # 即时期望奖励（A 台：0.6×1 + 0.4×(-1)）

for gamma in [0.0, 0.5, 0.9, 0.99, 0.999]:
    if gamma < 1:
        V = R / (1 - gamma)
    else:
        V = float("inf")
    print(f"γ = {gamma:<5}  =>  V = {V:.4f}")
```

:::output
γ = 0.0 => V = 0.2000
γ = 0.5 => V = 0.4000
γ = 0.9 => V = 2.0000
γ = 0.99 => V = 20.0000
γ = 0.999 => V = 200.0000
:::

$\gamma$ 越接近 1，$V$ 越大——智能体越"远视"，未来奖励的累积贡献越高。当 $\gamma = 0$ 时，$V$ 就等于单步期望奖励 0.2，完全不在乎未来。

也可以用迭代法逼近 $V$——从 $V = 0$ 开始，反复代入贝尔曼方程，直到收敛：

```python
gamma = 0.9
V = 0.0
for i in range(50):
    V_new = R + gamma * V
    V = V_new

print(f"迭代 50 次后 V = {V:.6f}")
print(f"理论值        V = {R / (1 - gamma):.6f}")
```

:::output
迭代 50 次后 V = 1.998698
理论值 V = 2.000000
:::

迭代 50 次后几乎精确收敛到理论值——这就是贝尔曼方程的收敛性在起作用。后面会看到，DP 方法正是利用这种迭代来求解价值函数的。

## 贝尔曼最优方程

贝尔曼期望方程评估的是某个给定策略 $\pi$ 的价值。但 RL 的终极目标不是评估策略，而是**找到最优策略 $\pi^*$**。

贝尔曼最优方程只做了一个改动：把"对所有动作取策略加权平均"换成"选最好的那个动作"：

$$V^*(s) = \max_{a} \left[ R(s, a) + \gamma \sum_{s' \in \mathcal{S}} P(s' | s, a) \, V^*(s') \right] \tag{3.3}$$

两个方程的对比：

|          | 贝尔曼期望方程                        | 贝尔曼最优方程                   |
| -------- | ------------------------------------- | -------------------------------- |
| **公式** | $\sum_a \pi(a\|s)[\cdot]$（加权平均） | $\max_a [\cdot]$（取最大值）     |
| **含义** | "如果我按策略 $\pi$ 行动，能拿多少分" | "如果我每步都选最优，能拿多少分" |
| **依赖** | 需要知道 $\pi$                        | 不需要 $\pi$                     |

为什么最优方程里没有 $\pi$？因为 $\max_a$ 直接选价值最高的动作，不需要显式定义"以多大概率选哪个动作"。只要知道了最优价值 $V^*$ 或 $Q^*$，最优策略就确定了：$\pi^*(s) = \arg\max_a Q^*(s, a)$。

### 用一个具体例子验证贝尔曼最优方程

考虑一个有 3 个状态的简单 GridWorld：

```
┌─────┬─────┬─────┐
│  s1 │  s2 │  s3 │
│     │     │  🏆 │
└─────┴─────┴─────┘
```

规则：每走一步扣 1 分，到达 $s_3$（宝藏处）游戏结束，奖励为 0。只能往右走，$\gamma = 1$。

从右往左推：$V^*(s_3) = 0$（已经到了终点），$V^*(s_2) = \max_a [R(s_2, \text{右}) + \gamma \cdot V^*(s_3)] = -1 + 1 \times 0 = -1$，$V^*(s_1) = \max_a [-1 + V^*(s_2)] = -1 + (-1) = -2$。

现在假设在 $s_1$ 有两个选择：向右（走到 $s_2$，扣 1 分）和向上（撞墙，留在 $s_1$，扣 2 分）。贝尔曼最优方程告诉你：向右的价值是 $-1 + V^*(s_2) = -1 + (-1) = -2$，向上的价值是 $-2 + V^*(s_1) = -2 + (-2) = -4$。$\max_a$ 选向右——显然是更好的选择。这就是贝尔曼最优方程的核心逻辑：**在每个状态都做最优选择，得到的就是全局最优。**

### 用 Python 验证 GridWorld

```python
import numpy as np

# 3 状态 GridWorld：s0 -> s1 -> s2 (终点)
n_states = 3
gamma = 1.0
step_cost = -1

# 动作：右 (stay_cost=-1)、上 (撞墙, cost=-2)
actions = {"右": -1, "上": -2}  # reward for each action

# 终点价值为 0，从右往左用贝尔曼最优方程递推
V_star = [0.0] * n_states

for s in range(n_states - 2, -1, -1):
    best = -float("inf")
    for name, r in actions.items():
        if name == "右":
            s_next = min(s + 1, n_states - 1)
        else:  # 撞墙
            s_next = s
        value = r + gamma * V_star[s_next]
        best = max(best, value)
        print(f"  s{s} 动作'{name}': {r} + {gamma}×{V_star[s_next]:.1f} = {value:.1f}")
    V_star[s] = best
    print(f"=> V*({s}) = {best:.1f}\n")

print("最优价值:", {f"s{i}": v for i, v in enumerate(V_star)})
```

:::output
s1 动作'右': -1 + 1.0×0.0 = -1.0
s1 动作'上': -1 + 1.0×-1.0 = -2.0
=> V\*(1) = -1.0

s0 动作'右': -1 + 1.0×-1.0 = -2.0
s0 动作'上': -1 + 1.0×-2.0 = -3.0
=> V\*(0) = -2.0

最优价值: {'s0': -2.0, 's1': -1.0, 's2': 0.0}
:::

代码在每个状态比较所有动作，取最大值——正是贝尔曼最优方程 $\max_a$ 的实现。结果和手动推导完全一致。

<details>
<summary>思考题：$\max_a$ 看起来简单，实际有什么问题？</summary>

$\max_a$ 要求对所有动作都计算一遍再取最大。当动作空间很大（比如大模型有 5 万个 token 作为动作候选）时，这个"取最大"操作本身就变成了一个困难的计算问题。这正是路线一（Value-Based）在面对连续或超大动作空间时遇到的核心瓶颈——也是路线二（Policy-Based）存在的理由之一。

</details>

## V 的局限：不告诉你选哪个动作

$V(s)$ 给了你评估局面的能力。但它留下两个缺口：

1. **怎么从数据中把 $V$ 估计出来？** 贝尔曼方程给出了理论公式，但你需要知道环境的转移概率 $P$。现实中几乎不可能——那该怎么办？下一节将介绍三种经典方法——DP、MC、TD——每一代都解决了前一代的核心局限。
2. **$V$ 不区分动作。** $V(s) = 80$ 只说明"这个棋局不错"，但不告诉你"该走车还是走马"。要做出决策，你还需要更多信息——这正是后面的 $Q(s, a)$ 要解决的问题。
