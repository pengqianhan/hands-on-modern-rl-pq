# MDP 五元组与价值函数

## 本节导读

**核心内容**

- 掌握 MDP 五元组的每个组成部分及其在老虎机、CartPole、LLM 对话中的含义。
- 理解折扣累积回报 $G_t$ 如何把多步奖励压成一个总目标。
- 学会区分状态价值 $V(s)$、动作价值 $Q(s,a)$，以及两者之间的关系。

**核心公式**

$$\mathcal{M} = (\mathcal{S}, \mathcal{A}, P, R, \gamma)$$

$$G_t = \sum_{k=0}^{\infty}\gamma^k r_{t+k}$$

$$V^\pi(s) = \mathbb{E}_\pi[G_t \mid s_t=s], \qquad Q^\pi(s,a) = \mathbb{E}_\pi[G_t \mid s_t=s, a_t=a]$$

$$V^\pi(s) = \sum_a \pi(a\mid s)Q^\pi(s,a)$$

上一节我们用两台老虎机体验了 RL 的基本交互模式：选择动作，获得奖励，循环往复。但体验归体验——你能直觉地说出"状态"到底是什么吗？"策略"和"奖励"之间是什么关系？为什么有的任务需要一个叫 $\gamma$ 的参数，有的却不需要？

这些问题需要一个统一的数学语言。这个语言就是马尔可夫决策过程（Markov Decision Process, MDP）。

MDP 框架的思想根源可以追溯到俄罗斯数学家安德烈·马尔可夫在 20 世纪初提出的"马尔可夫链"——一种描述随机过程如何演化的数学工具。到了 1950 年代，美国数学家理查德·贝尔曼在兰德公司研究多阶段决策问题时，发现马尔可夫链恰好能描述"每一步都面临选择"的场景。他在 1957 年的经典著作中系统化了这一框架 [^2]，从此 MDP 成为运筹学和最优控制的核心工具。半个世纪后， Sutton 和 Barto 把它继承到强化学习中，于是就有了我们今天看到的样子。

不管你的智能体在下围棋、控制机器人、还是和人类对话，底层都可以用一个 MDP 来建模。

## MDP 五元组：(S, A, P, R, γ)

一个 MDP 由五个要素定义，通常写作 $(S, A, P, R, \gamma)$。

你可能会觉得"五元组"听起来很抽象。别急——先看一张总览表建立直觉，然后我们逐个拆解，不仅说"是什么"，更要说"为什么这样设计"。

| 符号     | 名称     | 老虎机             | CartPole                    |
| -------- | -------- | ------------------ | --------------------------- |
| $S$      | 状态集合 | {初始}（单状态）   | $\mathbb{R}^4$（4维连续值） |
| $A$      | 动作集合 | {选 A, 选 B}       | {左推, 右推}                |
| $P$      | 转移概率 | 始终回到同一状态   | 由物理引擎决定              |
| $R$      | 奖励函数 | 中奖 +1，没中 -1   | 存活 +1                     |
| $\gamma$ | 折扣因子 | 不涉及（单步游戏） | 通常 0.99                   |

**S（状态集合）**——"环境此刻长什么样"。状态是环境在某一时刻的完整描述。在老虎机中，状态只有一个——不管你选择了多少次，每一轮面对的情况都一样，所以不需要区分不同的状态。在 CartPole 中，状态是一个 4 维向量 $[x, \dot{x}, \theta, \dot{\theta}]$，分别代表小车的位置、速度、杆子的角度和角速度。

为什么状态必须包含做决策所需的**所有**信息？这正是"马尔可夫"的含义——只要你掌握了当前的完整状态，过去的历史就不再提供额外的信息。就像下象棋：只要你看清了当前的棋盘，你不需要知道之前是怎么走到这个局面的，就能做出最优决策。反过来说，如果状态遗漏了关键信息——比如你只看到棋盘却不知道轮到谁走——那决策就会出错。这就是为什么 CartPole 不仅告诉你杆子的角度，还告诉你角速度：角度相同但正在加速倾斜的杆子，和正在恢复平衡的杆子，需要的应对完全不同。

**A（动作集合）**——"智能体能做什么"。老虎机中只有"选 A"和"选 B"两个离散动作。CartPole 也是两个离散动作（左推/右推）。而在连续控制任务中（如机器人行走），动作可以是实数向量（如关节力矩）。

你可能会问：为什么要把动作空间也写进 MDP 的定义里？因为不同的动作空间决定了算法的选择。离散动作空间天然适合 Q-learning 类方法——你只需要比较几个 $Q$ 值的大小。连续动作空间则更适合策略梯度方法——因为"从无穷多个动作中取 max"是不现实的，但"调整一个概率分布的参数"是可行的。这正是 PPO 在工业界被广泛采用的原因之一——它对两种动作空间都能胜任。

**P（转移概率）**——"世界如何运转"。$P(s' | s, a)$ 描述了：在状态 $s$ 下采取动作 $a$ 后，环境变成状态 $s'$ 的概率。在老虎机中，不管你选哪台，状态都不变——它始终停在"初始"状态。但在 CartPole 中，$P$ 由牛顿力学精确决定——施加一个力，小车就会按物理定律移动。

这里有一个非常重要的设计选择：在 MDP 的定义中，$P$ 只依赖于当前状态 $s$ 和动作 $a$，而不依赖于更早的历史。这正是马尔可夫性的数学表述——世界的运转规律是"无记忆"的。如果这个假设被打破了（比如股票价格的趋势效应），那 MDP 框架就不够用了，你需要更复杂的模型（如 POMDP——部分可观测 MDP）。好在大多数 RL 任务在合理的状态表示下都能近似满足马尔可夫性。

另一个关键区分：在老虎机中，$P$ 的规律（出奖率）是未知的——智能体需要通过交互来"摸索"。大多数真实 RL 问题也是如此。你不知道环境的 $P$，但这并不妨碍你定义 MDP——$P$ 客观存在，只是智能体不知道而已。

**R（奖励函数）**——"做得好不好"。$R(s, a)$ 告诉智能体这一步拿了多少分。奖励是 RL 中唯一的学习信号——Sutton 提出的"奖励假设"认为，所有目标都可以被描述为"最大化期望累积奖励信号"。

奖励函数的设计看似简单，实则是一个需要深思熟虑的问题。在老虎机中，奖励很直白——中奖 +1，没中 -1。但在大模型对齐中，"什么样的回答值得高分"本身就是一个研究课题——这正是 RLHF 和 DPO 要解决的核心问题。更微妙的是，奖励信号通常只反映结果的好坏，而不指明具体哪一步决策更优。智能体必须从多次交互中自行推断——这正是 RL 区别于监督学习的核心特征。

**γ（折扣因子）**——"未来值多少钱"。$\gamma \in [0, 1]$ 控制智能体对"延迟满足"的重视程度。$\gamma$ 接近 1 表示"未来和眼前差不多重要"，$\gamma$ 接近 0 表示"我只管眼前这一步"。在老虎机中，每轮只选择一次，所以不涉及"未来"。但在 CartPole 这样的多步游戏中，$\gamma$ 就很关键了——我们马上就会详细讨论。

为了加深直觉，让我们看看三种截然不同的 RL 任务的五元组：

| 要素     | 老虎机               | CartPole               | LLM 对话               |
| -------- | -------------------- | ---------------------- | ---------------------- |
| $S$      | {初始}（单状态）     | $\mathbb{R}^4$（连续） | 已生成的 token 序列    |
| $A$      | {选 A, 选 B}         | {左, 右}               | 词表中的下一个 token   |
| $P$      | 出奖率（固定但未知） | 物理引擎               | 语言模型自身的采样     |
| $R$      | ±1                   | +1/步                  | 人类偏好/奖励模型打分  |
| $\gamma$ | —（单步）            | 0.99                   | 通常不使用（整句评估） |

尽管表面的差异巨大，但它们都是 MDP——都用同一套数学语言描述。这就是 MDP 框架的强大之处。

<details>
<summary>思考题：LLM 生成满足马尔可夫性吗？</summary>

严格来说，LLM 的自回归生成不满足传统意义上的马尔可夫性——下一个 token 的概率不仅取决于当前 token，还取决于整个上文。但我们可以把"已生成的完整序列"定义为状态 $s$，这样状态转移就只依赖于当前状态（完整上文）和动作（下一个 token），从而满足马尔可夫性。

在实际操作中，RLHF 通常把整个生成过程看作一个 episode，在序列结束后才给予奖励。这更接近蒙特卡洛方法，而非逐步 TD 学习。

</details>

## 折扣累积回报 $G_t$

在老虎机中，每轮只选择一次，我们只关心单步的期望回报。但在 CartPole 这样的多步游戏中，智能体关心的不是某一步的奖励，而是**整个时间线上的总收益**。但未来的奖励不能和眼前的奖励一视同仁——就像今天的 100 块钱比明年的 100 块更有价值一样，未来的奖励需要打一个折扣。

这个类比不只是修辞，它背后有严谨的经济学基础。在金融学中，"今天的 100 块比明年的 100 块更值钱"有三个原因：通货膨胀让未来的购买力缩水；今天拿到钱可以投资获得利息；未来是不确定的——你不知道明年是否真能拿到这笔钱。RL 中的 $\gamma$ 捕捉了完全相同的三重含义：$\gamma$ 充当了"利率"的角色——未来的奖励需要按 $(1-\gamma)$ 的"利率"折现；同时它也反映了"未来不确定"的直觉——越远的事情越不可控，所以权重越低。

折扣累积回报 $G_t$ 的定义是：

$$G_t = r_t + \gamma \, r_{t+1} + \gamma^2 \, r_{t+2} + \gamma^3 \, r_{t+3} + \cdots$$

用一个具体例子来感受：假设 CartPole 每步奖励 +1，$\gamma = 0.9$，智能体活了 4 步后杆子倒了：

$$G_0 = \underbrace{1}_{r_0} + \underbrace{0.9 \times 1}_{r_1} + \underbrace{0.9^2 \times 1}_{r_2} + \underbrace{0.9^3 \times 1}_{r_3} = 1 + 0.9 + 0.81 + 0.729 = 3.439$$

同样是 +1 的奖励，但第 3 步的 1 分只值 $0.729$——因为 $\gamma = 0.9$ 连乘三次后，"未来的 1 分"贬值到不到原来的四分之三。$G_t$ 就是从现在开始到游戏结束，把所有奖励按"越远越不值钱"的规则累加起来的总分。

为什么不直接让 $\gamma = 1$？因为在无限期游戏中，如果 CartPole 永远活下去，总奖励会趋于无穷大——数学上就发散了。折扣因子让累积奖励收敛到一个有限值，同时也符合经济学中的时间偏好：当前的收益比未来的等量收益更有价值。

## 价值函数 V(s) 与 Q(s, a)

有了 MDP 的框架和 $G_t$ 的定义，我们终于可以回答一个核心问题：这个局面到底值多少分？

### 状态价值函数 $V^\pi(s)$

同一个状态、同一个策略，每次跑出来的 $G_t$ 可能不一样——因为环境的转移和策略本身都可能带有随机性。怎么办？取期望（平均值）。这就是状态价值函数 $V^\pi(s)$：

$$V^\pi(s) = \mathbb{E}_\pi \left[ G_t \mid s_t = s \right] = \mathbb{E}_\pi \left[ \sum_{k=0}^{\infty} \gamma^k \, r_{t+k} \;\middle|\; s_t = s \right]$$

$V^\pi(s)$ 的意思是：站在状态 $s$，遵循策略 $\pi$ 行动，平均能拿多少分。用一个类比来说，$V(s)$ 就像一个棋局评估器——给你一个棋盘局面，它告诉你从这步开始最终能赢的概率有多大。好的局面 $V$ 值高，差的局面 $V$ 值低。

在老虎机中，只有一个状态，策略是"永远选 A"（$\pi(\text{选 A}) = 1$），每步期望回报是 0.2，那么（不考虑折扣时）：

$$V = 0.6 \times (+1) + 0.4 \times (-1) = 0.2$$

这个 0.2 就是：站在起点，遵循"永远选 A"的策略，平均每次能赢 0.2 分。

注意 $V$ 的上标是 $\pi$——这意味着同一个状态，用不同的策略会得到不同的价值。好策略的 $V$ 高，差策略的 $V$ 低。RL 的终极目标，就是找到让 $V$ 最高的那个策略 $\pi^*$。

### 动作价值函数 $Q^\pi(s, a)$

$V^\pi(s)$ 告诉你"这个局面整体值多少分"。但有时候你需要的不是整体的评分，而是"走这步棋之后值多少分"——这就是 $Q$ 函数：

$$Q^\pi(s, a) = \mathbb{E}_\pi \left[ \sum_{k=0}^{\infty} \gamma^k \, r_{t+k} \;\middle|\; s_t = s, a_t = a \right]$$

和 $V$ 的唯一区别：$Q$ 多了一个条件 $a_t = a$——它固定了第一步的动作。继续用棋类来类比：

- $V(s)$：这个棋局整体胜率是多少？（不分走哪步）
- $Q(s, \text{出车})$：如果你走了出车这一步，之后的胜率是多少？
- $Q(s, \text{出马})$：如果你走了出马这一步，之后的胜率是多少？

显然，如果 $Q(s, \text{出车}) > Q(s, \text{出马})$，那你应该出车。比较不同动作的 $Q$ 值，就能做出更好的决策。

### V 和 Q 的关系

$V$ 和 $Q$ 之间有一个优雅的关系式。既然 $V(s)$ 是"不管走哪步整体值多少分"，而 $Q(s, a)$ 是"走某步之后值多少分"，那么 $V$ 自然就是所有 $Q$ 的加权平均——权重就是策略选择每个动作的概率：

$$V^\pi(s) = \sum_{a} \pi(a | s) \cdot Q^\pi(s, a)$$

用老虎机来验证。假设 A 台出奖率 60%，策略是"70% 选 A，30% 选 B"：

- $Q(s, \text{选 A}) = 0.6 \times 1 + 0.4 \times (-1) = 0.2$
- $Q(s, \text{选 B}) = 0.4 \times 1 + 0.6 \times (-1) = -0.2$
- $V(s) = 0.7 \times 0.2 + 0.3 \times (-0.2) = 0.14 - 0.06 = 0.08$

验算：每次 70% 概率选 A（期望 0.2）、30% 概率选 B（期望 -0.2），混合下来确实就是 0.08。状态的价值 = 所有动作价值的策略加权平均，就像一个班级的平均分等于各科成绩按学分加权平均一样。

这个数值验证说明了一件重要的事：**$V$ 和 $Q$ 不是两个独立的概念，而是同一枚硬币的两面。** $V(s)$ 是"在策略 $\pi$ 下，状态 $s$ 的整体预期"，$Q(s, a)$ 是"在状态 $s$ 选了动作 $a$ 之后的预期"。$V$ 是宏观视角，$Q$ 是微观视角。知道其中一个，就能算出另一个。这个互推关系在后续章节会反复出现——第 5 章的 Critic 网络输出 $V$，第 4 章的 DQN 学习 $Q$，但它们本质上在做同一件事：评估"这个局面值多少分"。

### 为什么价值函数这么重要？

因为 RL 的核心任务就是估计 $V$ 或 $Q$。一旦你准确知道了每个状态的 $V$ 值（或每个状态-动作对的 $Q$ 值），最优策略就显而易见了——选 $Q$ 值最大的那个动作。正如 Sutton 和 Barto 在他们的经典教材中所强调的：**几乎所有 RL 算法都可以理解为在估计价值函数，然后利用价值函数来改进策略。**

这两个函数将在后续章节反复登场：第 5 章的 Critic 网络就是 $V(s)$ 的神经网络实现，第 6 章的 GAE 是对 $Q - V$（优势函数）的精巧估计，第 4 章的 DQN 直接学习 $Q(s, a)$ 的神经网络。理解了 $V$ 和 $Q$，你就掌握了理解这些算法的钥匙。

<details>
<summary>思考题：如果 V(s) 对所有状态都是 0，意味着什么？</summary>

如果 $V(s) = 0$ 对所有 $s$ 成立，说明当前策略在所有状态下都"既不赚也不亏"。这可能有两种情况：一是策略是纯随机的，环境也是公平的（如两台完全一样的老虎机，随机选择）；二是策略太差了，以至于在好的局面和差的局面都拿不到正的回报。更一般地说，$V(s)$ 的相对大小比绝对值更重要——好状态的 $V$ 应该高于差状态的 $V$。

</details>

现在我们知道了价值函数是什么——但 $V(s)$ 的定义要求将当前时刻到终止的所有奖励累加起来，这在实际计算中并不可行。你不可能穷举未来无穷多步来算 $V$。有没有办法只看一步，就能算出价值？

这正是理查德·贝尔曼在 1950 年代苦苦思索的问题——而他的答案，就是我们下一节要学习的[贝尔曼方程](./bellman-equation)。如果你还记得刚才手填宝藏地图时的思路，你可能已经无意中"发明"了这个方程。
