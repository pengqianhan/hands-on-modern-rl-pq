# 经典方法速览：DP、MC 与 TD

## 本节导读

**核心内容**

- 掌握 DP、MC、TD 三类价值估计方法分别需要什么信息、何时更新、误差来自哪里。
- 理解从已知模型到采样轨迹，再到一步自举更新的演进逻辑。
- 学会用 TD Error 连接价值评估、Q-Learning、Critic 训练等后续算法。

**核心公式**

$$V(s) \leftarrow \sum_a \pi(a\mid s)\left[R(s,a)+\gamma\sum_{s'}P(s'\mid s,a)V(s')\right]$$

$$V(s) \leftarrow V(s)+\alpha\left[G_t-V(s)\right]$$

$$V(s) \leftarrow V(s)+\alpha\left[r+\gamma V(s')-V(s)\right]$$

$$\delta = r+\gamma V(s')-V(s)$$

上一节得到了贝尔曼方程——它给了计算 $V$ 的理论公式。但有个前提：你得知道环境的全部信息（转移概率 $P$、奖励函数 $R$）。CartPole 的物理模型你有，但围棋的 $10^{170}$ 个状态呢？大模型的 token 分布呢？现实中的 $P$ 和 $R$ 几乎总是未知的。

所以问题变成了：**不知道 $P$ 和 $R$，怎么从数据中把 $V$ 估计出来？** 有三代方法，它们不是教科书按时间排列的三个备选方案，而是一条演进线——每一代都解决了前一代的核心局限：

- **DP**：精确，但需要知道一切 → 现实中做不到
- **MC**：不需要模型，但要等一局结束才能更新 → 太慢
- **TD**：不需要模型，走一步就能更新 → 现代 RL 的主力

理解这条演进线，你就能理解为什么 PPO 的 Critic 用 TD 而不是 MC，为什么 DQN 要用经验回放池。这三代方法是后续所有算法的基因。

## 第一代：动态规划（Dynamic Programming, DP）

### 核心思想

如果你完全知道环境的转移概率 $P$ 和奖励函数 $R$，你可以直接用贝尔曼方程迭代地计算 $V$。这就是动态规划，由贝尔曼在 1957 年提出 [^1]。

你可能会问：什么情况下你会"完全知道" $P$ 和 $R$？答案是：当你自己设计了环境的时候。比如 GridWorld 的规则是你写的——你明确知道"向右走一步扣 1 分，从 (0,1) 一定到 (0,2)"。在这种场景下，DP 就是最好的选择——不需要采样，不需要探索，直接精确计算。这也是为什么 DP 在运筹学和最优控制（如库存管理、路径规划）中至今仍是主力工具——这些领域的问题通常有已知的模型。

### 更新规则

策略评估（Policy Evaluation）的迭代公式就是贝尔曼期望方程的直接实现：

$$V(s) \leftarrow \sum_a \pi(a|s) \left[ R(s,a) + \gamma \sum_{s'} P(s'|s,a) \, V(s') \right] \tag{3.4}$$

反复对所有状态执行这个更新，$V$ 最终会收敛到 $V^\pi$ 的精确值。

:::details 数学推导：这个公式从哪来？每一项是什么意思？

#### 从贝尔曼期望方程到迭代更新

上一节我们学过贝尔曼期望方程（公式 3.2）：

$$V^\pi(s) = \sum_a \pi(a|s) \left[ R(s,a) + \gamma \sum_{s'} P(s'|s,a) \, V^\pi(s') \right]$$

这是一个**等式**——左边等于右边。但问题是：等号两边都出现了 $V$，你不知道 $V$，就没法直接解。

DP 的想法是：**把等号变成赋值箭头**。先随便猜一个 $V$（比如全 0），然后反复用右边的公式计算新的 $V$，用新的替换旧的：

$$V_{k+1}(s) \leftarrow \sum_a \pi(a|s) \left[ R(s,a) + \gamma \sum_{s'} P(s'|s,a) \, V_k(s') \right]$$

下标 $k$ 表示第几轮迭代。可以证明，当 $k \to \infty$ 时，$V_k$ 收敛到 $V^\pi$（证明依据：贝尔曼算子是压缩映射，见下文）。

#### 逐项拆解

从外到内，公式有三层结构：

**第一层：$\sum_a \pi(a|s) [\cdots]$ — 策略加权**

策略 $\pi$ 告诉你在状态 $s$ 下以多大概率选每个动作。$\pi(a|s)$ 就是"选动作 $a$ 的概率"。$\sum_a$ 表示把所有动作的结果按概率加权求和。

比如：策略是"70% 往右，30% 往左"，那 $V(s) = 0.7 \times [\text{往右的价值}] + 0.3 \times [\text{往左的价值}]$。

**第二层：$R(s,a) + \gamma \sum_{s'} P(s'|s,a) V(s')$ — 一步的"总价值"**

这是"在状态 $s$ 选了动作 $a$ 之后"的期望总价值，由两部分组成：

- $R(s,a)$：这一步**立刻**拿到的奖励（确定性的）
- $\gamma \sum_{s'} P(s'|s,a) V(s')$：**未来**的期望价值——所有可能的下一个状态 $s'$，按转移概率 $P(s'|s,a)$ 加权，再乘以折扣因子 $\gamma$

**第三层：$P(s'|s,a)$ — 转移概率**

"在状态 $s$ 做了动作 $a$ 之后，转移到状态 $s'$ 的概率"。这是环境的规则——比如"在冰面上向右走，90% 真的向右，10% 滑到下面"。

#### 为什么用 $\gamma$（折扣因子）？

$\gamma < 1$ 有两个作用：

1. **数学上保证收敛**：每一轮迭代，$V$ 的变化量会被 $\gamma$ 缩小，最终收敛。如果 $\gamma = 1$，无限期任务的 $V$ 可能不收敛。
2. **直觉上"重眼前轻未来"**：未来的奖励不如眼前实在——$\gamma = 0.9$ 意味着 10 步之后的 1 分奖励，现在只值 $0.9^{10} \approx 0.35$ 分。

#### 为什么一定会收敛？（压缩映射定理）

定义贝尔曼算子 $T$：$TV(s) = \sum_a \pi(a|s) [R(s,a) + \gamma \sum_{s'} P(s'|s,a) V(s')]$。

可以证明 $\|TV_1 - TV_2\|_\infty \leq \gamma \|V_1 - V_2\|_\infty$（$\gamma < 1$，所以 $T$ 是压缩映射）。根据 Banach 不动点定理，反复迭代 $T$ 必然收敛到唯一的不动点 $V^\pi$。

**直觉**：每次迭代，旧估计与新估计之间的差距至少被缩小到 $\gamma$ 倍。$\gamma < 1$ 保证差距越缩越小，最终趋于 0。

#### 用微型走廊验证公式的每一步

3 格走廊 S → M → G，每步奖励 -1，$\gamma = 1$，确定性策略"只往右走"。

转移概率：$P(\text{M}|S,\text{右}) = 1$，$P(\text{G}|M,\text{右}) = 1$，其余为 0。

**迭代 0→1**：$V$ 全为 0。

- $V(S) = 1 \times [(-1) + 1 \times V(M)] = -1 + 0 = -1$
- $V(M) = 1 \times [(-1) + 1 \times V(G)] = -1 + 0 = -1$
- $V(G) = 0$（终点）

**迭代 1→2**：

- $V(S) = (-1) + V(M) = -1 + (-1) = -2$
- $V(M) = (-1) + V(G) = -1 + 0 = -1$

**迭代 2→3**：$V(S) = -1 + (-1) = -2$，不再变化。收敛！

你看到关键机制了吗？**V(M) 在第 1 轮更新为 -1 之后，立即参与了第 2 轮 V(S) 的计算**——这就是"全状态同步扫描"的威力，信息一步就从终点传到了起点。
:::

### 一个微型例子

假设一个 3 格走廊，S → M → G，每步奖励 -1，$\gamma = 1$。初始 $V$ 全部为 0。

| 迭代 | V(S) | V(M) | V(G)      |
| ---- | ---- | ---- | --------- |
| 0    | 0    | 0    | 0         |
| 1    | -1   | -1   | 0         |
| 2    | -2   | -1   | 0         |
| 3    | -2   | -1   | 0（收敛） |

经过 3 次迭代就收敛了——因为 V(M) 更新后立即影响了 V(S) 的下一步更新。这种"全状态扫描"的方式保证了快速收敛。

### 策略迭代（Policy Iteration）

策略评估只是第一步。知道了 $V^\pi$，就可以用它来改进策略——在状态 $s$ 选择让 $Q(s,a)$ 最大的动作：

$$\pi'(s) = \arg\max_a \left[ R(s,a) + \gamma \sum_{s'} P(s'|s,a) V^\pi(s') \right]$$

策略迭代是"评估策略 → 改进策略 → 再评估"的循环，理论上保证收敛到最优策略 $\pi^*$。

:::details 数学推导：策略改进为什么一定有效？

#### 策略迭代的两步循环

策略迭代由两个交替步骤组成：

1. **策略评估**（Policy Evaluation）：给定策略 $\pi$，用公式 3.4 迭代求解 $V^\pi$
2. **策略改进**（Policy Improvement）：用 $V^\pi$ 构造一个更好的策略 $\pi'$

关键问题是：**凭什么 $\pi'$ 比 $\pi$ 好？**

#### 策略改进定理（Policy Improvement Theorem）

定义 $Q^\pi(s, a) = R(s, a) + \gamma \sum_{s'} P(s'|s,a) V^\pi(s')$，这是"在状态 $s$ 选动作 $a$，之后按 $\pi$ 行动"的价值。

新策略 $\pi'$ 定义为贪心策略：

$$\pi'(s) = \arg\max_a Q^\pi(s, a)$$

策略改进定理说的是：**如果 $Q^\pi(s, \pi'(s)) \geq V^\pi(s)$ 对所有 $s$ 成立，那么 $V^{\pi'}(s) \geq V^\pi(s)$ 对所有 $s$ 成立**。

#### 证明思路

因为 $\pi'(s)$ 是 $\arg\max_a Q^\pi(s,a)$，所以：

$$Q^\pi(s, \pi'(s)) = \max_a Q^\pi(s, a) \geq Q^\pi(s, \pi(s)) = V^\pi(s)$$

展开 $Q^\pi(s, \pi'(s))$：

$$R(s, \pi'(s)) + \gamma \sum_{s'} P(s'|s,\pi'(s)) V^\pi(s') \geq V^\pi(s)$$

对右边的 $V^\pi(s')$ 再展开一层贝尔曼方程（用同样的 $\pi'$），得到 $V^{\pi'}$ 的表达式。反复展开，利用 $\gamma < 1$ 保证级数收敛，最终可得 $V^{\pi'}(s) \geq V^\pi(s)$。

#### 什么时候停止？

当 $\pi' = \pi$ 时（贪心策略不再改变），意味着对每个状态 $s$：

$$\pi(s) = \arg\max_a Q^\pi(s, a)$$

这正是贝尔曼最优方程的条件——$\pi$ 已经是最优策略 $\pi^*$。

#### 为什么不直接用贝尔曼最优方程？

你可能会问：贝尔曼最优方程 $V^*(s) = \max_a [R(s,a) + \gamma \sum_{s'} P(s'|s,a) V^*(s')]$ 里有个 $\max$，它是非线性的，不能像策略评估那样直接迭代求解（因为 $\max$ 不可微分，不满足压缩映射的条件）。策略迭代的巧妙之处在于：把 $\max$ 拆成"先评估一个固定策略"（线性，可迭代）+"再用 $\arg\max$ 改进策略"两步，每一步都是简单操作。
:::

### 局限

现实中几乎不可能知道完整的 $P$ 和 $R$。围棋有 $10^{170}$ 个状态，LLM 有天文数字的 token 序列组合。DP 的精确性建立在对环境的完全知识之上，这使它更像一个理论基准——告诉你"知道一切时的最优答案"。

## 第二代：蒙特卡洛方法（Monte Carlo, MC）

### 核心思想

不知道环境模型？那就跑起来看看。MC 方法以摩纳哥赌城命名——因为它本质上就是"靠运气采样"。这个名字最早由乌拉姆和冯·诺伊曼在曼哈顿计划中使用，后被引入 RL 领域 [^2]。MC 的核心思想：采样完整的轨迹（从起点到终点），用实际回报 $G_t$ 来估计 $V(s)$。

### 更新规则

$$V(s) \leftarrow V(s) + \alpha \left[ G_t - V(s) \right] \tag{3.5}$$

$\alpha$ 是学习率。$G_t - V(s)$ 是"实际拿到的分数减去之前的预测"——差多少就补多少。

:::details 数学推导：MC 更新公式从哪来？

#### 从"样本平均"到"增量更新"

MC 的核心思想：用样本回报的平均值来估计 $V(s)$。

假设状态 $s$ 被访问了 $N$ 次，每次得到的回报分别是 $G_1, G_2, \ldots, G_N$。最直接的做法：

$$V(s) = \frac{1}{N} \sum_{i=1}^{N} G_i$$

但问题是：你每次得到新的 $G_{N+1}$，就要重新算一遍所有 $N+1$ 个值的平均——既浪费内存又浪费时间。能不能只保留当前的 $V(s)$，来一个新的 $G$ 就更新一次？

**增量平均公式**：设当前平均为 $V_N$，新来一个样本 $G_{N+1}$，新的平均是：

$$V_{N+1} = \frac{1}{N+1} \sum_{i=1}^{N+1} G_i = \frac{1}{N+1} \left( N \cdot V_N + G_{N+1} \right) = V_N + \frac{1}{N+1} (G_{N+1} - V_N)$$

关键步骤推导：

$$V_{N+1} = \frac{N \cdot V_N + G_{N+1}}{N+1} = \frac{N \cdot V_N + V_N - V_N + G_{N+1}}{N+1} = V_N + \frac{G_{N+1} - V_N}{N+1}$$

这个公式告诉你：**新估计 = 旧估计 + 步长 × (新样本 - 旧估计)**。

#### 从 $\frac{1}{N+1}$ 到 $\alpha$

在严格的增量平均中，步长是 $\frac{1}{N+1}$——随着样本增多自动变小，保证收敛。

但在 RL 中，我们通常用一个固定的学习率 $\alpha$ 来替代 $\frac{1}{N+1}$：

$$V(s) \leftarrow V(s) + \alpha \left[ G_t - V(s) \right]$$

为什么？

- **好处**：$\alpha$ 固定意味着智能体永远不会"停止学习"——即使环境发生变化（比如老虎机的中奖概率突然变了），$\alpha > 0$ 保证智能体能持续跟踪变化。
- **代价**：$V$ 永远不会完全收敛到真实值，而是在真实值附近以 $O(\alpha)$ 的幅度波动。
- **实际中**：通常取 $\alpha = 0.01$ 或 $\alpha = 0.1$，或用衰减的 $\alpha_t = \frac{1}{t}$。

#### $G_t - V(s)$ 的含义：预测误差

$G_t - V(s)$ 就是"实际回报与预测价值的差"：

- $G_t > V(s)$：实际比预测好 → $V$ 上调
- $G_t < V(s)$：实际比预测差 → $V$ 下调
- $G_t = V(s)$：预测准确 → 不动

这与人类学习的直觉完全一致——你预测考试能考 80 分（$V(s)$），实际考了 90 分（$G_t$），下次你就会调高对自己的评估。

#### 为什么 MC 是无偏的？

"无偏"的定义：$\mathbb{E}[V(s)] = V^\pi(s)$，即估计值的期望等于真实值。

MC 用的是真实的完整回报 $G_t = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \cdots$——这是从状态 $s$ 出发，实际跑完整个 episode 拿到的总回报。根据 $V^\pi(s)$ 的定义：

$$V^\pi(s) = \mathbb{E}[G_t | s_t = s]$$

所以 $G_t$ 就是 $V^\pi(s)$ 的无偏估计（它的期望恰好等于 $V^\pi(s)$），用 $G_t$ 的平均值来估计 $V$ 自然也是无偏的。

#### 为什么方差大？

$G_t = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \cdots$ 把从 $t$ 时刻到 episode 结束的**所有随机奖励**加在了一起。每一步的奖励都是随机的，加在一起，不确定性就累加了——就像掷 100 次骰子的总和，比掷 1 次骰子的方差大得多。

形式化地：如果每步奖励 $\text{Var}(r_i) = \sigma^2$，则（粗略地）$\text{Var}(G_t) \approx \frac{\sigma^2}{1-\gamma^2}$，随路径长度指数级增长。
:::

### 特点

MC 方法给出的是**无偏估计**——因为你用的是真实的完整回报 $G_t$，不包含任何估计值。但方差巨大：同一个状态，不同 episode 的 $G_t$ 波动很大。

以老虎机为例：策略"永远选 A"，跑 10 个 episode。由于 60% 的出奖率是随机的，这 10 个 episode 的回报可能是 [0.2, -0.1, 0.3, 0.1, -0.2, 0.4, 0.0, 0.2, -0.1, 0.3]，平均值 0.11——离真实的 0.2 还有距离。需要跑很多 episode 才能收敛。

什么叫"方差大"？用更具体的数字来感受。假设你跑 5 个 episode，分别得到回报 [0.4, -0.2, 0.6, 0.0, 0.8]，平均 0.32。再跑 5 个，得到 [-0.1, 0.5, 0.1, -0.3, 0.3]，平均 0.10。两组的平均值相差 0.22——这就是高方差：每次估计的结果都很不稳定。要收敛到真实值 0.2 附近（比如误差在 ±0.02 以内），你可能需要跑上千个 episode。"无偏"的意思是，跑无穷多次后均值一定收敛到真实值；"高方差"的意思是，有限次采样的结果波动剧烈，你需要非常多的样本才能得到可靠的估计。

### 局限

必须等到 episode 结束才能更新——不能边走边学。对于没有自然终点的连续任务（如机器人持续工作），MC 的适用性受限。

## 第三代：时序差分学习（Temporal Difference, TD）

### 核心思想

TD 学习由 Sutton 和 Barto 在 1988 年正式提出 [^3]，被他们称为"RL 核心的、新颖的想法"。TD 是 DP 和 MC 的折中——既不需要环境模型（像 MC），又能边走边学（不像 MC 要等结束），同时还利用了自举（bootstrapping，像 DP）。

什么是"自举"（bootstrapping）？就是用一个估计去更新另一个估计——TD 用 $V(s')$ 来更新 $V(s)$，而 $V(s')$ 本身也是一个估计值，不是真实值。这听起来像是"拔着自己的鞋带把自己提起来"——用一个不准确的估计去修正另一个不准确的估计，真的靠谱吗？答案是：虽然每一步都有偏差（因为你用的是估计值而非真实值），但经过足够多的更新，偏差会逐步缩小，最终收敛。这正是 TD 的巧妙之处：它牺牲了无偏性（引入了偏差），换来了大幅降低的方差和在线学习的能力。

### 更新规则

$$V(s) \leftarrow V(s) + \alpha \underbrace{\left[ r + \gamma V(s') - V(s) \right]}_{\text{TD Error } \delta} \tag{3.6}$$

方括号里的 $r + \gamma V(s')$ 叫做 **TD Target**——"实际结果 + 对未来的估计"，也就是你"本该"预测的值。$V(s)$ 是你之前的预测。两者之差就是 TD Error。

:::details 数学推导：TD 更新公式从哪来？为什么比 MC 方差低？

#### 从贝尔曼方程到 TD 更新

回忆贝尔曼期望方程：$V^\pi(s) = \mathbb{E}[r + \gamma V^\pi(s') | s]$。

这个等式说的是：$V^\pi(s)$ 的真实值等于"$r + \gamma V(s')$ 的期望"。

DP 的做法：精确计算期望 $\sum_a \pi(a|s) [\cdots \sum_{s'} P(s'|s,a) \cdots]$——需要知道 $P$。

TD 的做法：**不计算期望，而是采样一次**。实际走一步，观测到 $(r, s')$，用这一个样本来近似期望：

$$V(s) \leftarrow V(s) + \alpha \left[ \underbrace{r + \gamma V(s')}_{\text{TD Target}} - V(s) \right]$$

这就是用**单步采样**替代了 DP 的**全概率求和**——不需要知道 $P$，只需要跑一步看看发生了什么。

|                        | MC                                                       | TD                    |
| ---------------------- | -------------------------------------------------------- | --------------------- |
| **目标**               | $G_t = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \cdots$ | $r + \gamma V(s')$    |
| **用到多少真实数据？** | 从 $t$ 到终止的所有奖励                                  | 只有一步的 $r$        |
| **"未来"怎么处理？**   | 全部用真实奖励                                           | 用估计值 $V(s')$ 代替 |

TD Target 只用了一步的真实奖励 $r$，然后用 $V(s')$ 来"代替"未来所有的奖励——这就是**自举（bootstrapping）**。

#### 为什么 TD 有偏差？

TD 的目标 $r + \gamma V(s')$ 中，$V(s')$ 是一个估计值，不是真实值。所以：

$$\mathbb{E}[r + \gamma V(s')] \neq V^\pi(s)$$

除非 $V(s') = V^\pi(s')$（即估计已经完全正确），否则 TD Target 的期望不等于真实值——这就是偏差的来源。

相比之下，MC 的目标 $G_t$ 是真实回报，$\mathbb{E}[G_t] = V^\pi(s)$——无偏。

#### 为什么 TD 方差更低？

$G_t$ 把所有步骤的随机性都叠在一起了（$k$ 步的随机奖励求和），而 TD Target 只包含**一步**的随机性（$r$ 是随机的，但 $V(s')$ 在更新时是固定的）。

形式化地：

- $\text{Var}(G_t) \propto$ 轨迹长度 × 单步方差（随路径长度增长）
- $\text{Var}(r + \gamma V(s')) \propto$ 单步方差（与路径长度无关）

这就是 TD 的核心权衡：**牺牲无偏性（引入偏差），换取大幅降低的方差**。

#### TD(0) 的收敛性

虽然单步更新有偏差，但可以证明 TD(0) 在满足以下条件时收敛：

1. $\sum_{t=0}^{\infty} \alpha_t = \infty$（总学习量足够大）
2. $\sum_{t=0}^{\infty} \alpha_t^2 < \infty$（学习率衰减足够快）

直觉：每次更新虽然不完美（有偏差），但更新方向在期望上是对的（TD Error 的期望趋近于 0）。足够多的更新之后，偏差逐步消除。

#### 一个具体数值例子

3 格走廊 S → M → G，真实价值 $V(S) = -2, V(M) = -1, V(G) = 0$。

假设当前估计 $V(S) = 0, V(M) = 0, V(G) = 0$，学习率 $\alpha = 0.5$。

**MC 方式**：从 S 出发跑完一局，得到轨迹 S→M→G，回报 $G = -1 + (-1) = -2$。

$V(S) \leftarrow 0 + 0.5 \times (-2 - 0) = -1$

只跑了一局，估计从 0 跳到 -1。再跑一局可能得到 -2（估计又大幅变化）——波动大。

**TD 方式**：从 S 出发走一步，观测到 $(r=-1, s'=\text{M})$。

$V(S) \leftarrow 0 + 0.5 \times (-1 + 1 \times 0 - 0) = -0.5$

走一步就更新了，变化幅度小。接着从 M 走一步到 G：

$V(M) \leftarrow 0 + 0.5 \times (-1 + 1 \times 0 - 0) = -0.5$

多跑几轮后，$V$ 会逐步逼近 $[-2, -1, 0]$。每步变化幅度小——方差低。
:::

### TD Error 的定义

$$\delta = r + \gamma V(s') - V(s) \tag{3.7}$$

:::details 数学推导：TD Error 的深层含义——贝尔曼方程的"残差"

#### TD Error 是什么？

TD Error 是贝尔曼方程在单步上的"违反程度"。如果把贝尔曼方程写成：

$$V^\pi(s) - \left[ R(s,a) + \gamma V^\pi(s') \right] = 0$$

当 $V$ 完全正确时，左边恒为 0。但在学习过程中 $V$ 还不准确，实际采样到 $(r, s')$ 后：

$$\delta = r + \gamma V(s') - V(s)$$

$\delta$ 就是"实际发生的事情"（$r + \gamma V(s')$）与"你的预测"（$V(s)$）之间的差距。

#### TD Error 的期望就是贝尔曼残差

对 $\delta$ 取期望（对所有可能的 $(r, s')$）：

$$\mathbb{E}[\delta] = \mathbb{E}[r + \gamma V(s') | s] - V(s) = \left[\sum_a \pi(a|s) \left( R(s,a) + \gamma \sum_{s'} P(s'|s,a) V(s') \right)\right] - V(s)$$

当 $V = V^\pi$ 时，右边方括号就是贝尔曼期望方程，等于 $V^\pi(s)$，所以 $\mathbb{E}[\delta] = 0$。

**$\mathbb{E}[\delta] = 0$ 等价于 $V$ 满足贝尔曼方程**——这就是为什么 TD Error 是"学习完成"的指标。

#### TD Error vs 梯度下降的关系

TD 更新可以写成：

$$V(s) \leftarrow V(s) + \alpha \cdot \delta$$

如果用神经网络参数化 $V_\theta$，更新变为：

$$\theta \leftarrow \theta + \alpha \cdot \delta \cdot \nabla_\theta V_\theta(s)$$

这看起来很像梯度下降，但它**不是真正的梯度下降**——因为 TD Target $r + \gamma V(s')$ 中的 $V(s')$ 也依赖于 $\theta$，而 TD 方法在计算梯度时假装 TD Target 是常数。这种"半梯度"方法在实践中工作得很好，但理论分析更复杂。

#### 三种误差信号对比

| 方法 | 误差信号                  | 无偏？                                    | 方差                 |
| ---- | ------------------------- | ----------------------------------------- | -------------------- | ----------- |
| DP   | $\sum_a \pi(a             | s) [R + \gamma \sum_{s'} P V(s')] - V(s)$ | 无偏（精确计算期望） | 0（非随机） |
| MC   | $G_t - V(s)$              | 无偏（真实回报）                          | 高                   |
| TD   | $r + \gamma V(s') - V(s)$ | 有偏（$V(s')$ 是估计）                    | 低                   |

DP 精确但需要模型，MC 无偏但方差大，TD 低方差但有偏差——三种方法分别占据"精确性"和"数据效率"的不同位置。

#### TD Error 贯穿全书的三重角色

1. **价值更新的驱动信号**：$V(s) \leftarrow V(s) + \alpha \delta$（本节）
2. **优势函数的最简形式**：$\hat{A}(s,a) = \delta = r + \gamma V(s') - V(s)$，衡量"这个动作比平均水平好多少"（第 6 章）
3. **GAE 的构建基石**：$A^{\text{GAE}} = \sum_{l=0}^{\infty} (\gamma \lambda)^l \delta_{t+l}$，多个 TD Error 的指数衰减加权和（第 7 章）
   :::

- $\delta > 0$：实际比预期好，价值被低估，应该上调 $V(s)$
- $\delta < 0$：实际比预期差，价值被高估，应该下调 $V(s)$
- $\delta = 0$：预测完美，学习完成

TD Error 不仅仅是一个误差指标——它在 RL 中承担着三重关键角色，贯穿后续几乎所有算法：

1. **Critic 的训练信号**：Critic 网络直接用 $\delta$ 来更新价值估计（第 6 章）
2. **最简优势函数**：$\delta$ 本身就是对"做了这个动作比预期好多少"的估计（第 6 章）
3. **GAE 的基础**：GAE 是多个 TD Error 的指数加权和（第 7 章）

### TD vs MC：一个直觉对比

想象你预测明天降雨概率是 80%（$V(s) = 0.8$）。第二天实际没下雨（$r = 0$），而且你观察到后天也不太会下雨（$V(s') = 0.3$）。

- TD 视角：$\delta = 0 + 0.9 \times 0.3 - 0.8 = -0.53$。预测偏高，下调。
- MC 视角：需要等到整个天气周期结束才能回头修正，而且那次周期可能跨越很多天，$G_t$ 的波动很大。

TD 的优势在于：**走一步就能修正预测，不需要等结束**。

:::details 思考题：TD Error 能不能永远为 0？
在确定性环境中，理论上可以：只要所有状态的 $V$ 值都精确满足贝尔曼方程，TD Error 就处处为 0。但在随机环境和函数逼近（神经网络）的情况下，TD Error 通常不会精确为 0。因为环境的随机性导致每次采样到的 $(r, s')$ 不同，而神经网络的参数有限，无法精确表示所有状态的价值。所以在实际训练中，我们追求的是 TD Error 的期望趋近于 0。
:::

## 三代方法的演进

|                    | **DP**               | **MC**           | **TD**                 |
| ------------------ | -------------------- | ---------------- | ---------------------- |
| **需要环境模型？** | 是                   | 否               | 否                     |
| **需要完整轨迹？** | 否                   | 是               | 否                     |
| **自举**           | 是                   | 否               | 是                     |
| **偏差**           | 无                   | 无               | 有（但低方差）         |
| **方差**           | 低                   | 高               | 中                     |
| **代表场景**       | 已知规则的小规模问题 | 有终止的采样任务 | 绝大多数无模型在线任务 |

演进逻辑：DP 精确但不现实 → MC 不需要模型但要等太久 → TD 不需要模型也不需要等。每一代都解决了前一代的核心局限。

学完这一节，你获得了一项关键能力：**从数据中估计价值函数**。DP 是你的理论基准（"如果知道一切会怎样"），MC 告诉你"纯靠数据能做到什么"，TD 是你的实际工具（"数据有限、边做边学时用什么"）。三者共享同一个核心——贝尔曼方程——只是"怎么拿到未来信息"的方式不同。

如果你是初学者，应该从哪个开始？答案是 **TD**。不是因为 TD 最简单，而是因为它是现代 RL 的主力——PPO、SAC、TD3 等工业界广泛使用的算法都以 TD 为基础。理解了 TD Error 和自举的思想，你就能直接理解第 5 章的 Actor-Critic 和第 6 章的 GAE。DP 更适合作为理论基准来理解"完美情况下会怎样"，MC 则帮助你理解"不借助任何估计，纯靠数据能做什么"。三条线都值得了解，但 TD 是你的主战场。

### MC→TD 演进在策略空间的重现

这条演进线不仅出现在价值估计中。在第 5 章，我们将看到完全相同的演进出现在策略优化中：

| 价值估计                    | 策略优化                    | 方法                   |
| --------------------------- | --------------------------- | ---------------------- |
| MC（用 $G_t$）              | REINFORCE（用 $G_t$）       | 跑完一整局，用完整回报 |
| TD（用 $r + \gamma V(s')$） | Actor-Critic（用 $\delta$） | 走一步就更新           |

REINFORCE 就是策略版的 MC，Actor-Critic 就是策略版的 TD。理解了价值估计中 MC→TD 的演进，后续理解策略优化中的对应演进就会水到渠成。
