# 第3章：MDP与价值函数——从猜硬币到贝尔曼方程

> **本章目标**：在前两章动手经验的基础上，系统引入 RL 的数学框架。你将理解"为什么 RL 能学会东西"背后的形式化表述。

## 3.1 动手：玩一个简化 MDP 游戏——猜硬币正反面

- 状态：当前硬币面（正/反）；动作：猜正/猜反；奖励：猜对 +1，猜错 -1
- 写一个随机策略跑 100 轮，记录总回报
- **你将获得**：在极简环境中亲手体验"状态-动作-奖励"循环

## 3.2 观察与疑问

- 为什么随机策略的长期期望回报接近 0？
- 如果硬币有偏（正面 60%），最优策略是什么？回报会变成多少？
- 如果加入折扣因子 γ=0.9，"未来奖励"还值多少钱？
- **你将思考**：策略的好坏取决于环境结构（转移概率）和目标设定（折扣因子）

## 3.3 理论：MDP 五元组 (S, A, P, R, γ)

- **S（状态集合）**：环境可能处于的所有局面。用 CartPole 举例：4 维连续状态空间
- **A（动作集合）**：智能体可以采取的所有操作。CartPole：{左, 右}；猜硬币：{猜正, 猜反}
- **P（转移概率）**：做了动作后环境如何变化。CartPole 中由物理引擎决定；猜硬币中由硬币偏度决定
- **R（奖励函数）**：每一步获得的即时反馈信号。CartPole：+1（存活）；猜硬币：±1
- **γ（折扣因子）**：未来奖励的"贬值率"。γ 接近 1 = 重视长远；γ 接近 0 = 只顾眼前
- 用 CartPole 每一步拆解五元组的具体含义
- **你将理解**：MDP 是所有 RL 问题的统一数学描述，不管问题是下棋还是对话

## 3.4 价值函数 V(s) 与 Q(s, a)

- **状态价值函数 V(s)**："当前局面值多少分"——从状态 s 出发，遵循某策略，未来期望累积奖励
- **动作价值函数 Q(s, a)**："当前局面做某个动作值多少分"——在状态 s 选择动作 a 后的期望累积奖励
- V 与 Q 的关系：V(s) = Σ π(a|s) · Q(s, a)
- 直觉类比：V(s) = 这个棋局的胜率；Q(s, a) = 走这步棋之后的胜率
- **你将理解**：RL 的核心任务就是估计 V 或 Q，然后根据它们做决策

## 3.5 贝尔曼方程

- **贝尔曼期望方程**：V(s) = R(s) + γ Σ P(s'|s,a) V(s')——"当前的价值 = 即时奖励 + 折扣后的未来价值"
- **贝尔曼最优方程**：V\*(s) = max_a [R(s,a) + γ Σ P(s'|s,a) V\*(s')]——"最优价值 = 选最好动作后的价值"
- 两个方程的区别：前者是对给定策略的评估，后者是寻找最优策略的工具
- **你将理解**：贝尔曼方程把"评估一个局面"变成递归问题——当前价值依赖于未来价值

## 3.6 回看实践：手算猜硬币游戏的 V 值

- 手算：偏硬币（正 60%）下，一直猜正的 V 是多少？
  - V = 0.6×(+1) + 0.4×(-1) + γ × V → 解方程得 V = 0.2 / (1-γ)
  - 当 γ=0.9 时，V = 2.0
- 用代码验证贝尔曼方程在仿真中成立
- **你将获得**：从手算到代码验证，确认"理论推导"和"实际运行"一致
