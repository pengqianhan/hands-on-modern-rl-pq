# 第4章：策略梯度——从摇骰子到REINFORCE

> **本章目标**：理解"如何用梯度下降直接优化策略"这一核心思想。这是 PPO、GRPO 等现代算法的共同理论基础。

## 4.1 动手：摇骰子赌博机

- 环境：两个动作——摇 A（赢率 30%，赢 +1）、摇 B（赢率 70%，赢 +1）
- 用 REINFORCE 训练一个策略网络（单层 Softmax），看它如何学会偏爱 B
- 观察策略概率从 0.5/0.5 逐渐变成 0.1/0.9
- **你将获得**：亲眼看到"梯度更新"如何让策略从"两个都试试"变成"专挑好的"

## 4.2 观察训练曲线

- 策略概率的演化过程：先慢后快，最终收敛
- 为什么有震荡？——因为每次采样只有有限数据，梯度估计有噪声
- 如果学习率太大？策略会在 A 和 B 之间剧烈摇摆
- **你将思考**：策略梯度的核心挑战——高方差

## 4.3 理论：策略梯度的数学推导

- 目标函数：J(θ) = E_π[ Σ γ^t r_t ]——策略的期望累积奖励
- 策略梯度定理：∇J(θ) = E[ Σ ∇log π(a|s) × G_t ]
  - G_t = 从时刻 t 到结束的累积奖励（回报）
  - 直觉解读："如果一个动作导致了好的结果，就增加再做这个动作的概率"
- 为什么用 log 概率而不是直接用概率？（数值稳定性 + 正确的梯度方向）
- **你将理解**：策略梯度的核心思想——"好的结果 → 强化对应动作的概率"

## 4.4 REINFORCE 算法

- 蒙特卡洛采样更新：跑完一整个 episode，用实际回报 G_t 作为"这个动作好不好"的估计
- REINFORCE 的完整流程：
  1. 用当前策略采样一条轨迹
  2. 计算每步的回报 G_t
  3. 用 G_t × ∇log π(a_t|s_t) 更新参数
- 方差问题：G_t 的波动非常大——同样的动作，不同 episode 可能得到截然不同的回报
- 为什么需要基线？——减去一个"平均回报"可以消除"运气"带来的噪声
- **你将理解**：REINFORCE 是最朴素的策略梯度算法，原理清晰但实际效果不稳定

## 4.5 Actor-Critic 架构

- **Actor（演员）**：负责选择动作——输出策略 π(a|s)
- **Critic（评论家）**：负责评估局面——输出价值估计 V(s) 或 Q(s,a)
- 优势函数：A(s,a) = Q(s,a) - V(s)——"这个动作比平均水平好多少"
- Actor-Critic 的更新流程：
  1. Actor 选动作，环境反馈奖励和新状态
  2. Critic 评估"这个转移值多少分"
  3. 用 Critic 的评估指导 Actor 的更新
- 为什么比 REINFORCE 好？Critic 提供了更稳定的"好坏评判"基准
- **你将理解**：Actor-Critic 是后续所有主流算法（PPO、SAC、TD3）的骨架结构

## 4.6 回看实践：给摇骰子加上基线

- 对比实验：有基线 vs 无基线的训练曲线
- 无基线：曲线震荡剧烈，收敛慢
- 有基线（减去平均回报）：曲线平滑，收敛快
- 解读：基线减掉了"运气"带来的方差，只保留"真正因为动作选择带来的好坏差异"
- **你将获得**：直观理解"方差"和"基线"这两个贯穿全书的核心概念
