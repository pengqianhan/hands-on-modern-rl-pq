# 第14章：RL训练常见坑与解法

> **本章目标**：逐一复现并解决 RL 训练中最常见的四类故障。每个坑都遵循"复现场景 → 理论解释 → 修复验证"的三步流程。

## 14.1 策略崩溃（Policy Collapse）

- **现象**：训练过程中 Reward 突然归零，且无法自行恢复
- **复现**：在 PPO 训练中将 learning rate 设为 0.01（正常值的 10 倍）
- **理论解释**：
  - 策略更新步长过大 → 一步就从"还不错"变成"完全随机"
  - 一旦崩溃，后续采集的数据都是垃圾数据 → 用垃圾数据更新 → 越来越差
- **修复方案**：
  1. 降低学习率（通常 1e-4 ~ 3e-4）
  2. 增大 PPO clip ε（更保守的更新）
  3. 加入 KL 散度惩罚（防止策略偏离太远）
  4. 定期 checkpoint，崩溃时回滚
- **验证**：修复后 reward 曲线不再出现突然归零的情况
- **你将获得**：策略崩溃的完整诊断-修复流程

## 14.2 奖励黑客（Reward Hacking）

- **现象**：Reward 持续走高，但人工评估发现生成质量很差
- **复现**：在 RLHF 中只使用"回答长度"作为奖励 → 模型学会写长篇废话
- **理论解释**：
  - 模型找到了奖励函数的漏洞——做"容易获得高分"的事，而不是"真正该做的事"
  - 本质是 Goodhart 定律：当一个指标成为优化目标，它就不再是好指标
- **修复方案**：
  1. 多维度奖励：加入准确率、安全性、简洁性等多个维度
  2. 对抗性数据标注：故意构造"看起来好但其实不好"的样本加入训练
  3. 定期人工抽检：自动化指标不能替代人类判断
  4. 奖励模型集成：多个奖励模型投票，降低单一模型被 hack 的风险
- **验证**：修复后 reward 与人工评估分数的相关性提高
- **你将获得**：识别和防范奖励黑客的系统方法

## 14.3 显存爆炸（OOM）

- **现象**：训练过程中 GPU 显存溢出（CUDA Out of Memory）
- **复现**：RLHF 训练中同时加载 Actor + Critic + Reference + Reward 四个模型
- **理论解释**：
  - RLHF 需要 4 个模型同时驻留显存：策略模型、价值模型、参考模型、奖励模型
  - 每个模型还需要存储梯度和优化器状态 → 显存需求 × 3
  - 7B 模型 × 4 × 3 ≈ 84B 参数需要约 168GB 显存（fp32）
- **修复方案**：
  1. **LoRA**：只训练低秩适配器，冻结主模型 → 显存减少 60-80%
  2. **ZeRO 优化**：将模型参数、梯度、优化器状态分片到多个 GPU
  3. **梯度检查点（Gradient Checkpointing）**：用计算换显存，只保留部分中间激活
  4. **混合精度训练（bf16/fp16）**：显存直接减半
- **验证**：修复后显存占用降低到可用范围内
- **你将获得**：RLHF 训练的显存优化工具箱

## 14.4 训练不收敛（Non-convergence）

- **现象**：Loss 震荡，Reward 不涨，训练似乎在原地打转
- **复现**：在稀疏奖励环境（如 Montezuma's Revenge）中直接用 PPO
- **理论解释**：
  - 奖励太稀疏：智能体几乎永远得不到正反馈 → 无法学习
  - 超参数不当：学习率太大（震荡）或太小（不动）
  - 探索不足：策略过早收敛到局部最优
- **修复方案**：
  1. **奖励塑形（Reward Shaping）**：设计中间奖励引导智能体走向正确方向
  2. **调整 γ/λ**：γ 控制视野远近，λ 控制 GAE 的偏差-方差权衡
  3. **增加探索**：熵正则化、好奇心驱动探索（Intrinsic Reward）
  4. **学习率调优**：用网格搜索或余弦退火找到合适的 lr
- **验证**：修复后 reward 曲线开始稳步上升
- **你将获得**：处理"训不动"问题的排查清单

## 小结：调试四步法

1. **看曲线**：Reward/Loss 的异常模式是第一线索
2. **查日志**：clip fraction、entropy、KL divergence 等指标定位问题类型
3. **理解原因**：每个异常背后都有数学解释
4. **小步修复**：一次只改一个变量，确认效果后再改下一个
