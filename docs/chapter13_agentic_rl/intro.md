# 第13章：Agentic RL——工具使用的自主进化

> **本章目标**：理解如何用 RL 训练智能体学会使用工具。你将掌握动作空间扩展、多轮交互建模等 Agentic RL 的核心技术。

## 13.1 动手：训练一个会用计算器的智能体

- 环境：数学题（如"124 × 37 + 89 = ?"）+ 可选的 `<calculator>` 工具调用动作
- 初始状态：模型要么直接猜答案，要么调用计算器
- 训练目标：让模型学会在需要精确计算时主动调用工具
- **你将获得**：观察模型从"什么都自己算（经常算错）"到"知道什么时候该求助工具"

## 13.2 观察训练过程

- 工具调用频率 vs 准确率：
  - 训练初期：几乎不用工具，准确率低
  - 训练中期：开始频繁调用工具，准确率上升
  - 训练后期：只在需要时调用工具，准确率稳定在高位
- 工具调用时机是否正确？
  - 正确行为：复杂计算调工具，简单心算不调
  - 错误行为：每道题都调工具（滥用）或从来不调（忽视）
- **你将思考**：好的工具使用 = 知道什么时候用、什么时候不用

## 13.3 理论：Agentic RL 的核心思想

- **动作空间扩展**：
  - 传统 RL：动作 = {左, 右} 或连续力矩
  - Agentic RL：动作 = {生成文本, 调用工具 A, 调用工具 B, ...}
  - 挑战：动作空间变成混合类型（文本 + 结构化工具调用）
- **轨迹级奖励分配（Trajectory-level Reward）**：
  - 问题：工具调用本身不产生奖励，只有最终结果才有奖励
  - 解决：用整个轨迹的成功/失败来更新每一步的策略
  - 类比：一场比赛的胜负不能只归功于最后一球
- **你将理解**：Agentic RL 的核心难题——如何让模型"学会选择工具"而不是"记住固定流程"

## 13.4 工具调用环境设计

- **定义工具 API 与观测反馈**：
  - 工具接口：`<calculator> expression </calculator>` → 返回计算结果
  - 工具接口：`<search> query </search>` → 返回搜索摘要
  - 模型输出被解析：检测是否包含工具调用 tag → 执行并返回结果
- **多轮交互的 MDP 建模**：
  - 状态 = (原始问题, 已生成的文本, 工具返回结果)
  - 动作 = 下一个 token（可能是工具调用的开始）
  - 终止 = 生成结束标记或达到最大长度
- **你将理解**：Agentic RL 的环境设计比传统 RL 复杂得多——需要处理文本解析和工具执行

## 13.5 回看实践：分析训练曲线

- 工具调用率上升与准确率提升的同步性——理想情况是两者同步上升
- 是否有"滥用工具"现象？——检查简单题（如 "2+3=?"）是否也在调工具
- 是否有"忽视工具"现象？——检查复杂题（如 "789×456=?"）是否仍然自己猜
- 工具调用位置的合理性——是否在推理的关键节点调用，而不是随机位置
- **你将获得**：评估 Agentic RL 训练效果的实用检查清单
