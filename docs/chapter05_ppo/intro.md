# 第5章：PPO——稳定训练的艺术

> **本章目标**：深入理解当前使用最广泛的 on-policy 算法 PPO。你将掌握裁剪机制和 GAE 两大核心组件，并理解 PPO 如何从游戏控制延伸到 LLM 对齐。

## 5.1 动手：用 PPO 训练 LunarLander

- 运行现成实现，观察着陆器从爆炸到平稳降落
- 记录训练过程中的平均 reward、策略熵、价值损失、clip fraction
- **你将获得**：PPO 的完整训练日志长什么样，哪些指标值得重点关注

## 5.2 观察与疑问

- Reward 曲线为何会突然跳水又恢复？
- 熵（Entropy）下降意味着什么？——从"到处探索"到"专注好动作"
- Clip fraction 过高时发生了什么？——策略在"剧烈摇摆"
- **你将思考**：RL 训练的稳定性不是免费的，需要精心的算法设计

## 5.3 理论：策略更新的危险

- 朴素策略梯度的问题：一步更新太大 → 策略崩溃（忘了之前学的一切）
- 重要性采样（Importance Sampling）：用旧策略收集的数据来更新新策略
- 策略比率 r_t(θ) = π_new(a|s) / π_old(a|s)——衡量新旧策略的差异
- **你将理解**：为什么不能无限制地重复利用旧数据更新策略

## 5.4 PPO 的裁剪机制

- PPO 的核心目标函数：L = min( r_t × A_t, clip(r_t, 1-ε, 1+ε) × A_t )
- 裁剪的效果：
  - 当优势 A_t > 0（好动作）：r_t 被限制不超过 1+ε，防止"好的就贪心过头"
  - 当优势 A_t < 0（坏动作）：r_t 被限制不低于 1-ε，防止"坏的就矫枉过正"
- 为什么 min 操作能防止破坏性更新？——取更保守的那个估计
- ε 的典型值：0.1~0.2，过大则更新太保守，过小则保护不足
- **你将理解**：PPO 的核心思想就一句话——"更新可以，但别更新太猛"

## 5.5 GAE：广义优势估计

- 问题：如何准确估计优势函数 A(s,a)？
- 时序差分（TD）估计：偏差大、方差小
- 蒙特卡洛（MC）估计：偏差小、方差大
- GAE = 在两者之间的连续插值：Â_t = Σ (γλ)^k δ_k
- λ 的意义：λ=0 退化为 TD（高偏差），λ=1 退化为 MC（高方差），通常取 0.95
- **你将理解**：偏差-方差权衡是 RL 中无处不在的核心张力

## 5.6 回看实践：分析 LunarLander 训练日志

- 找出 clip fraction 过大的时刻 → 对应 reward 跳水
- 调整 ε 和 λ，观察稳定性变化：
  - ε 从 0.2 降到 0.1 → 更新更保守，训练更稳定但可能更慢
  - λ 从 0.95 升到 0.99 → 方差增大，曲线更震荡
- **你将获得**：PPO 超参数调优的实战经验

## 5.7 PPO 在 LLM 对齐中的对应物

- Actor = 语言模型（生成回答）
- Critic = 价值模型（评估回答的好坏）
- 裁剪防止语言能力崩坏——不会因为一次 RL 更新就"忘了怎么说中文"
- KL 散度惩罚 = 额外的安全带，防止模型偏离原始分布太远
- **你将理解**：游戏环境中的 PPO 和 LLM 对齐中的 PPO，核心机制完全一致
