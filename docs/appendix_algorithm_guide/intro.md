# 附录 C：算法选型与工程框架

> **本附录目标**：学完第 3 章到第 11 章，你已经掌握了 DQN、PPO、DPO、GRPO、SAC、TD3 等算法。面对实际项目时，如何选择最合适的算法？本附录从动作空间、奖励结构、计算资源等维度出发，给出系统化的选型框架。

## 什么时候需要这份指南？

你在第 2 章用 DPO 跑通了对齐实验，第 6 章用 PPO 训练了 LunarLander，第 8 章用 GRPO 做了推理增强。每个算法都跑通了，但它们各自适用的场景截然不同。用错算法，轻则浪费算力，重则根本跑不出结果。

本附录把选型问题拆成两个专题文件：

| 文件                                  | 核心内容                                                     | 解决的问题                       |
| ------------------------------------- | ------------------------------------------------------------ | -------------------------------- |
| [算法选型决策](./algorithm-selection) | 决策速查表 + 五维度选型框架 + 常见选型错误                   | "面对我的问题，该用哪个算法？"   |
| [三层架构与 MBRL](./framework-mbrl)   | Rollout-Scoring-Training 三层架构 + 框架对比 + 基于模型的 RL | "怎么把算法变成可跑的工程系统？" |

第一个文件帮你选对算法，第二个文件帮你选对框架。两者缺一不可。
