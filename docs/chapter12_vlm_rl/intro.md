# 第12章：VLM强化学习——让视觉模型学会推理

> **本章目标**：将 RL 技术从纯文本扩展到视觉-语言多模态场景。你将理解 VLM RL 的独特挑战和前沿方法。

## 12.1 动手：用 GRPO 训练 VLM 回答视觉问题

- 数据集：简单几何图形（三角形/圆形/正方形）+ 数量问题（"图中有几个圆形？"）
- 奖励设计：答案正确 +1，推理步骤完整 +0.5，格式规范 +0.2
- 训练前后对比：模型从"瞎猜答案"到"先描述图片内容再给出推理"
- **你将获得**：VLM RL 的完整训练体验——与纯文本 GRPO 的核心流程一致，但输入多了图像

## 12.2 理论：VLM RL 的特殊挑战

- **视觉 token 与文本 token 的奖励分配**：
  - 一个回答由"看图"（视觉 token）和"推理"（文本 token）两部分组成
  - 奖励信号应该归因于视觉理解还是文本推理？——目前通常用整体奖励
- **多模态输入的策略表示**：
  - 策略网络需要同时处理图像编码器（ViT）和文本解码器（Transformer）
  - RL 更新会影响哪些部分？全量更新 vs 冻结视觉编码器
- **视觉幻觉问题**：模型可能学会"编造"图片中不存在的内容来凑高分
- **你将理解**：VLM RL ≠ 文本 RL + 图片，多模态带来了新的技术挑战

## 12.3 代表性框架

- **VisPlay：提问者 + 推理者自进化**
  - 提问者（Questioner）：生成有挑战性的视觉问题
  - 推理者（Reasoner）：回答视觉问题
  - 两者通过 RL 互相促进：问题越难 → 推理越强 → 问题生成更刁钻
- **VISTA-Gym：工具集成环境**
  - 将 Python 解释器、搜索引擎等工具纳入 VLM 的动作空间
  - 模型可以"写代码画图来验证自己的推理"
- **你将获得**：了解 VLM RL 的前沿研究方向

## 12.4 自动驾驶中的 VLM-RL

- 用 VLM 生成驾驶决策的奖励信号：
  - 输入：前方路况图像
  - VLM 判断："前方有行人，应该减速" → 生成对应奖励
- 安全约束下的策略优化：
  - 硬约束：某些动作（闯红灯）永远不能被奖励
  - 软约束：舒适度、效率等可以灵活权衡
- **你将理解**：VLM RL 在真实世界应用中的潜力与挑战

## 12.5 回看实践：分析 VLM 训练指标

- 视觉注意力热力图的变化：训练前注意力分散 → 训练后注意力集中在关键区域
- 推理步骤长度与准确率的关系：适度长度的推理效果最好（太短 = 没推理，太长 = 过度思考）
- 跨泛化测试：在训练集之外的新图形上的表现
- **你将获得**：VLM RL 训练效果的多维度评估方法
